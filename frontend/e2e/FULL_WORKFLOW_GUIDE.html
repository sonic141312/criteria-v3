1Tạo Schema (Cấu trúc dữ liệu)

Schema định nghĩa cấu trúc dữ liệu đầu vào cho evaluation. Ví dụ: đánh giá TikTok Creator.

Điều hướng

  • Vào trang Schemas từ sidebar
  • Click nút + New

Thông tin Schema

  • Tên: TikTok Creator Evaluation
  • Mô tả: Đánh giá và xếp hạng TikTok Creator
  • Submit form

Thêm Fields (Trường dữ liệu)

Followers

Key: followers
Display: Follower Count
Type: number

Likes

Key: likes
Display: Total Likes
Type: number

Shares

Key: shares
Display: Total Shares
Type: number

Views

Key: views
Display: Total Views
Type: number

Comments

Key: comments
Display: Total Comments
Type: number

Engagement Rate

Key: engagement_rate
Display: Engagement Rate
Type: number

💡 Mẹo: Mỗi field có key unique trong schema. Key sẽ được dùng trong các plugin input để đọc dữ liệu.

2Tạo Evaluation

Evaluation là phiên bản đánh giá gắn với một Schema. Tự động tạo Version 1 (DRAFT).

Điều hướng

  • Vào trang Evaluations từ sidebar
  • Click nút New Evaluation

Thông tin Evaluation

  • Chọn Schema: TikTok Creator Evaluation
  • Tên: Creator Ranking Evaluation
  • Mô tả: Xếp hạng Creator theo metrics
  • Submit form → Tự động chuyển sang Graph Builder
💡 Mẹo: Evaluation được tạo sẽ tự động tạo Version 1 ở trạng thái DRAFT. Chỉ version DRAFT mới có thể chỉnh sửa graph.

3Xây dựng Graph (Node & Edges)

Graph Builder cho phép thiết kế luồng xử lý đánh giá bằng cách kết nối các plugins.

Các loại Plugin

📥 Input 📊 Normalize ⚖️ Weighted Average 🔣 Formula ⚡ Threshold 📤 Output

Luồng xử lý mẫu (TikTok Creator)

Input: followers → Normalize 0-100 → Weighted Average → Threshold 70 → Output

Hướng dẫn thao tác

  • Thêm Node: Click nút Add Node hoặc kéo từ palette bên trái
  • Chọn loại plugin (Input, Normalize, Formula, etc.)
  • Nhập Label và Config tương ứng
  • Click Create
  • Kết nối Node: Kéo từ output port (bên phải) đến input port (bên trái)
  • Di chuyển Node: Drag node trên canvas
  • Zoom/Pan: Dùng chuột hoặc touchpad
  • Xóa Node/Edge: Click vào node/edge, chọn delete

4

1. Input Plugin 📥

Đọc giá trị từ dữ liệu đầu vào (inputValues khi execute).

  • Label: Tên hiển thị (VD: "Followers Input")
  • Config → fieldKey: Key của field trong schema (VD: "followers")
Output Port
value (number)

2. Normalize Plugin 📊

Chuyển đổi giá trị từ range này sang range khác.

  • Label: "Normalize Followers"
  • Config:
    • min: 0 (giá trị nhỏ nhất đầu vào)
    • max: 1000000 (giá trị lớn nhất đầu vào)
    • outMin: 0 (giá trị nhỏ nhất đầu ra)
    • outMax: 100 (giá trị lớn nhất đầu ra)
Ví dụ: followers = 500000 → normalized = 50

3. Weighted Average Plugin ⚖️

Tính trung bình có trọng số từ 2-5 đầu vào.

  • Label: "Main Score Calculator"
  • Config → weights: Mảng trọng số (tổng = 1.0)
    • [0.3, 0.25, 0.2, 0.15, 0.1] cho 5 inputs
Input Ports
a (required), b (required), c (optional), d (optional), e (optional)

4. Formula Plugin 🔣

Tính toán biểu thức phức tạp với AST.

  • Label: "Engagement Score"
  • Config → ast: Cấu trúc AST biểu thức
AST cho: base_score * 0.8 + bonus
{
  "type": "BinaryOp",
  "op": "+",
  "left": {
    "type": "BinaryOp",
    "op": "*",
    "left": { "type": "Variable", "name": "base_score" },
    "right": { "type": "Literal", "value": 0.8 }
  },
  "right": { "type": "Variable", "name": "bonus" }
}

Operators: +, -, *, /, %, >, <, >=, <=, ==, !=, &&, ||

5. Threshold Plugin ⚡

Phân loại dựa trên so sánh với ngưỡng.

  • Label: "Tier Classifier"
  • Config:
    • threshold: 70
    • comparison: "gte" (greater than or equal)
    • aboveOrEqualValue: "Premium Creator"
    • belowValue: "Standard Creator"

Comparisons: gte, gt, lt, lte, eq

6. Output Plugin 📤

Node cuối cùng, định nghĩa kết quả trả về.

  • Label: "Final Result"
  • Config → fields: Mảng tên các trường kết quả
    • ["finalScore", "tier"]

5Validate Graph

Trước khi publish, hệ thống kiểm tra graph để đảm bảo không có lỗi.

Các lỗi có thể phát hiện

🚫 Cycle Detected

Graph có chu trình (A→B→C→A). DAG required.

🔗 Orphan Node

Node không có kết nối vào/ra.

❌ Missing Input

Required port chưa được kết nối.

⚠️ Type Mismatch

Port không tương thích về kiểu dữ liệu.

📭 No Output

Graph thiếu Output node.

🚧 Unreachable Output

Output không thể truy cập từ input.

Thao tác

  • Click nút Validate
  • Xem kết quả: "Valid" (xanh) hoặc danh sách lỗi
  • Sửa lỗi nếu có và validate lại

6Publish Version

Publish để kích hoạt version. Khi publish, version cũ đã PUBLISHED sẽ trở thành ARCHIVED.

Trạng thái Version

  • DRAFT - Có thể chỉnh sửa
  • PUBLISHED - Đang hoạt động, có thể execute
  • ARCHIVED - Không hoạt động

Thao tác

  • Đảm bảo graph đã validate thành công
  • Click nút Publish
  • Xác nhận publish
  • Status sẽ chuyển từ DRAFT sang PUBLISHED
💡 Mẹo: Để tạo version mới, vào Version Manager. Version mới sẽ clone từ version PUBLISHED gần nhất.

7Run Execution

Thực thi evaluation với dữ liệu đầu vào cụ thể.

Điều hướng

  • Vào trang Executions từ sidebar

Chạy Execution

  • Chọn Evaluation
  • Chọn Version (thường là version đã PUBLISHED)
  • Nhập input values (định dạng JSON)
Input Values cho TikTok Creator
{
  "followers": 500000,
  "likes": 250000,
  "shares": 15000,
  "views": 1500000,
  "comments": 8000,
  "engagement_rate": 8.5
}

Result

  • Kết quả trả về từ Output node
  • Status: SUCCESS, PARTIAL, FAILED
Expected Result
{
  "finalScore": 78.5,
  "tier": "Standard Creator"
}

8Xem Execution Trace

Trace hiển thị chi tiết từng bước xử lý của graph.

Điều hướng

  • Từ trang Executions, click vào execution bất kỳ
  • Hoặc truy cập /executions/{id}/trace

Thông tin hiển thị

  • Danh sách tất cả nodes đã thực thi
  • Với mỗi node:
    • Giá trị đầu vào (inputsReceived)
    • Giá trị đầu ra (value)
    • Thời gian xử lý (durationMs)
    • Trạng thái (SUCCESS, SKIPPED, FAILED)
  • Thứ tự thực thi (topological sort)
  • Giải thích chi tiết cho từng node (explanation)
💡 Mẹo: Dùng Trace để debug khi kết quả không như mong đợi. Kiểm tra inputsReceived của từng node để xem dữ liệu có đúng không.

📋Quản lý Version (Version Manager)

Vào Version Manager từ Evaluation detail page để quản lý các version.

Tính năng

  • Xem danh sách tất cả versions
  • Xem trạng thái (DRAFT/PUBLISHED/ARCHIVED)
  • Tạo version mới (clone từ PUBLISHED gần nhất)
  • Publish version DRAFT
  • Xem metadata: ngày tạo, ngày publish
  • Navigate đến Graph Builder của từng version

Nodes trong Graph

  • 5 Input nodes: followers, likes, shares, views, engagement_rate

  • 1 Formula: Tính bonus từ engagement
  • 1 Threshold: Phân loại Premium/Standard
  • 1 Output: Trả về finalScore và tier
